# LED_DEMO_RTOS

STM32F103ZET6 LED blinking demo based on STM32 HAL, FreeRTOS, and CMSIS-RTOS v2.

## Project

- MCU: STM32F103ZET6
- Toolchain: GNU Arm Embedded Toolchain (`arm-none-eabi-gcc`)
- RTOS API: CMSIS-RTOS v2 over FreeRTOS
- Build system: Makefile generated by STM32CubeMX

## LED Pins

| LED | GPIO |
| --- | --- |
| LED0 | PB5 |
| LED1 | PE5 |

## RTOS Tasks

The application creates two CMSIS-RTOS v2 threads in `Core/Src/freertos.c`:

| Task | Entry function | Period |
| --- | --- | --- |
| `led0Task` | `StartLed0Task` | 500 ms |
| `led1Task` | `StartLed1Task` | 1000 ms |

Each task toggles one LED and then calls `osDelay()`, so LED blinking runs under the FreeRTOS scheduler instead of the `main()` infinite loop.

## Build

Make sure `arm-none-eabi-gcc` is available in `PATH`, then run:

```sh
make
```

The build outputs are generated in `build/`:

- `build/LED_DEMO_RTOS.elf`
- `build/LED_DEMO_RTOS.hex`
- `build/LED_DEMO_RTOS.bin`

To clean the build output:

```sh
make clean
```

## Flash

Connect the board through ST-LINK using SWD, then flash the ELF file:

```sh
STM32_Programmer_CLI -c port=SWD -w build/LED_DEMO_RTOS.elf -v -rst
```

## Notes

- The FreeRTOS configuration is in `Core/Inc/FreeRTOSConfig.h`.
- `configCHECK_FOR_STACK_OVERFLOW` is set to `2`.
- `configUSE_MALLOC_FAILED_HOOK` is enabled.
- `configASSERT()` is defined in the user code section of `FreeRTOSConfig.h`.
